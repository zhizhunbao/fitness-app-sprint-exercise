# FitLife Sprint Planning Dashboard

一个现代化的 Sprint Planning 静态页面项目，为 FitLife 健身管理应用的敏捷开发规划而设计。

## 项目概述

本项目为 Algonquin College 的 Sprint Planning 课堂练习创建了一个完整的演示页面，包含：

- 📊 **用户故事管理** - 5个详细的用户故事及其子任务分解
- 👥 **团队分工** - 4名团队成员的角色分配和技能展示
- 📈 **容量规划** - 可视化的容量对比承诺工作量
- 📋 **任务看板** - Todo/In Progress/Done 三栏看板展示
- 📉 **燃尽图** - Sprint 进度跟踪和团队速度分析
- 🎯 **交付物管理** - 功能性、文档性、流程性三类交付物
- 🔄 **Sprint 回顾** - 回顾分析和改进建议
- ⚠️ **风险管理** - 风险识别和缓解策略
- 🌐 **国际化支持** - 中英文双语切换

## 快速开始

### 运行项目

1. 克隆或下载项目到本地
2. 使用本地服务器运行（推荐）：

```bash
# 使用 Python
cd src
python -m http.server 8000

# 使用 Node.js
cd src
npx http-server

# 使用 Live Server (VS Code 扩展)
# 右键点击 src/index.html -> "Open with Live Server"
```

3. 在浏览器中访问 `http://localhost:8000`

### 数据修改

- 修改 `src/data/en.json` 来更新英文数据内容
- 修改 `src/data/zh.json` 来更新中文数据内容
- 页面会自动重新渲染更新的数据

## Sprint 规划详情

### Sprint 目标
开发用户仪表板和目标设置模块，为 FitLife 健身应用建立核心的用户交互基础。

### 时间安排
- **Sprint 持续时间**: 2周
- **团队规模**: 4名成员
- **总容量**: 315小时
- **已承诺工作**: 240小时
- **缓冲时间**: 24% (75小时)

### 故事点统计
- **总故事点**: 60点
- **P1 高优先级**: 34故事点 (用户档案创建、目标设置、基础仪表板)
- **P2 中优先级**: 26故事点 (活动记录、进度报告)
- **团队速度**: 52故事点/Sprint (94.5%完成率)

## 用户故事概览

### 高优先级 (P1)
1. **用户档案创建** (8点) - 创建包含个人信息的用户档案
2. **目标设置模块** (13点) - 设置健身目标的完整流程
3. **基础用户仪表板** (13点) - 显示档案和目标概览的仪表板

### 中优先级 (P2)
4. **简单活动记录** (13点) - 基础运动活动记录功能
5. **每周进度报告** (13点) - 自动生成的进度跟踪报告
